Detailed Description

MTC Client Interface Functions.

The MTC client startup procedure like below:
Mtc_CliInit --- init system resource
Mtc_CliOpen --- open an user for config provision
Using Mtc_CliDbSetLocalIp etc db APIs for set provision
Mtc_CliStart --- start client service
The MTC client halt procedure like below:
Mtc_CliStop --- stop client service
Mtc_CliClose --- close client user
Mtc_CliDestroy --- destroy system resource.
The MTC client for support multi-user switch procedure like below:
Mtc_CliStop --- stop client service
Mtc_CliClose --- close current user provision
Mtc_CliOpen --- open another user config provision
Using Mtc_CliDbSetLocalIp etc db APIs for set provision
Mtc_CliStart --- start client service

If Zos_CfgGetModDrvMode value is ZTRUE, it means MTC and MSF are running in different threads, User should implement callback which was set by Mtc_CliCbSetEvnt and post event to UI thread. The UI thread should Mtc_CliDrive explicitly after receive event.

If Zos_CfgGetModDrvMode value is ZFALSE, it means MTC and MSF are running in the same thread, MTC will call Mtc_CliDrive implicitly.

◆ Mtc_CliDrive()

ZFUNC ZINT Mtc_CliDrive ( ZEVNT  zEvntId)

Client Drive Service.

MTC service was implemented by MSF, for multi-thread mode, MSF is running in independent thread, GUI is another one generally. MTC implmented service login in GUI, so it is running in GUI. When MSF need notify status to GUI, the event should transfered in two threads. For thread safety, service event was posted by GUI callback Mtc_CliCbSetEvnt from MSF to GUI. Then GUI drive MTC by this service event.

Parameters
[in]zEvntIdService Event from RSD.
Return values
ZOKDrive Service successfully.
ZFAILEDDrive Service failed.

◆ Mtc_CliRefresh()

ZFUNC ZINT Mtc_CliRefresh ( ZFUNC_VOID  )

Refresh the registration.

This interface is try to send re-REGISTRE message to refresh the registration information on the server. Although the SDK will send re-REGISTER automatically accroding to the expire time length, there still be in some condition which the client wants to send re-REGISTER to keep registration information valid. Like on mobile phone platfrom, the SDK may not be able to send re-REGISTER on time during background mode. In these cases, the application should invoke this interface to send re-REGISTER manually to discover errors maybe occured during background mode. This interface must only be invoked after client has logined, Mtc_CliCbSetLclLoginOk.

Return values
ZOKClient send re-REGISTER ok.
ZFAILEDClient send re-REGISTER failed.
