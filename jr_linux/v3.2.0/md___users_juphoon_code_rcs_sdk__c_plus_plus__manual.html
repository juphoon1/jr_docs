这些文件的作用如下：

  • 3rdParty: 文件夹，存放JR SDK所依赖的三方库以及头文件。
  • ring: 铃音文件，包含响铃和挂断的铃声。
  • CMakeLists.txt: 完整工程的CMakeLists.txt, 内部会调用jrdemo和jrsdk的CMakeLists.txt
  • jrdemo: JR SDK 集成的demo, 其中 CMakeList.txt 为其编译文件。
  • jrsdk: JR SDK, 其中 CMakeList.txt 为其编译文件。
  • client_vers.sh, 生成 jrsdk的版本信息的shell脚本。

3. JR SDK 模块

JR SDK 目前分为下述模块：

  1. JRClient 模块: 涉及文件有，JRClient.*, JRClientImpl.*, JRClientCallback.*。该模块的主要提供Juphoon IMS SDK 初始化功能， 以及账号的注册功能。
    • JRClient.*: 基类，用于对外暴露接口。
    • JRClientImpl.*: JRClient 的实现，不对外提供。
    • JRClientCallback.*: JRClient的回调接口，用于通知上层, Juphoon IMS SDK的初始化状态以及客户端的注册状态, 由demo侧将其进行实现。
  2. JRAutoConfig 模块: 涉及文件有，JRAutoConfig.*, JRAutoConfigImpl.*, JRAutoConfigCallback.*。该模块的主要提供Juphoon IMS SDK自动配置相关的功能。
    • JRAutoConfig.*: 基类，用于对外暴露接口。
    • JRAutoConfigImpl.*: JRAutoConfig的实现, 不对外提供。
    • JRAutoConfigCallback.*: 自动配置相关的回调，用于通知上层, Juphoon IMS SDK的自动配置的情况, 由demo侧进行实现。
  3. JRAccount模块: 涉及文件有, JRAccount.*, JRAccountImpl.*。该模块为账号的创建以及进行配置设置。
    • JRAccount.*: 基类，用于对外暴露接口。
    • JRAccountImpl.*: 用于JRAccount的实现, 不对外暴露。
  4. JRCall模块: 涉及文件有, JRCall.*， JRCallImpl.*, JRCallCallback.*等文件。该模块主要提供Juphoon IMS SDK通话相关的功能。
    • JRCall.*: 基类, 用于对外暴露接口。
    • JRCallImpl.*: JRCall的实现，不对外提供。
    • JRCallCallback.*: 通话相关的回调，用于通知上层, Juphoon IMS SDK的通话状况, 由demo侧进行实现。
  5. JRNetworkRequest模块: 涉及文件有JRNetworkRequest.cc。该模块主要提供数据上报的功能，该部分接口不对外提供.
  6. JRMediaDevice模块: 涉及文件有JRMediaDevice.cc。 该模块主要是对ims sdk 媒体部分接口的封装，提供音频的开启与关闭的功能。

4. 编译

  1. 在当前目录下创建build目录，使得编译产生的临时文件都在build目录下，不会对源代码造成污染。
  2. 进入build目录下, 执行 cmake .. && make && make install。
  3. 编译会生成一个vowifi文件夹，内部包好有bin, include, lib三个文件夹。其中bin文件夹中包含有jrdemo的验证程序，include, 和lib文件夹用于客户自行使用集成。
  4. 添加环境变量 LD_LIBRARY_PATH, 其值为库所在的路径。例如执行命令export LD_LIBRARY_PATH=../lib/x86_64(该命令仅适用于x86_64环境)。
  5. 运行jrdemo, 执行./jrdemo

6. 集成

Juphoon IMS SDK 初始化

首先需要进行Juphoon IMS SDK的初始化, 相关接口见JRClient.h。 在初始化之前需要设置 JRClient 的回调进行监听,

{c++}
/**
* 添加回调。
*
* @param callback JRClientCallback 接口对象。
*/
void addCallback(JRClientCallback *callback)

初始化SDK接口调用

{c++}
/**
* @brief 初始化。
*
* @param appKey 用户从能力平台上申请的 AppKey 字符串。
* @param appId 应用ID。
* @param cuei 音箱设备号。
* @param vendor 应用名称。
* @param vendorName 音箱名称。
* @param vendorVersion 音箱版本号。
*/
void startInitWithAppkey(string appKey, string appId, string cuei,
string vendor, string vendorName, string vendorVersion);

初始化的结果通过onClientInitResult进行通知

{c++}
/**
* sdk 初始化结果回调。
*
* @param result 初始化 sdk 是否成功。
* - true：初始化成功。
* - false：初始化失败。
* @param reason 初始化 sdk 失败原因。参见 @ref JRClient.SUCCESS "初始化失败原因"。
*/
void onClientInitResult(bool result, int reason);

登录状态变更的通知

在程序运行过程中，所有有关注册状态变更的事件会通过JRClientCallback中的回调接口进行通知，具体 如下,

{c++}
/**
* 登录结果回调。
*
* @param result 登录是否成功。
* - true：登录成功。
* - false：登录失败。
* @param reason 登录失败原因。参见 @ref JRClient.REASON_NONE "登录失败原因"。
*/
virtual void onClientLoginResult(bool result,int reason) = 0;
/**
* 登出回调。
*
* @param reason 登出原因。参见 @ref JRClient.REASON_NONE "登出原因"。
*/
virtual void onClientLogoutResult(int reason) = 0;
/**
* 登录状态变化通知。
*
* @param state 当前登录状态值。参见 @ref JRClient.STATE_NOT_INIT "登录状态"。
* @param oldState 之前的登录状态值。参见 @ref JRClient.STATE_NOT_INIT "登录状态"。
*/
virtual void onClientStateChange(int state, int oldState) = 0;

通话过程中，JRSDK的状态变更通过回调的形式进行通知，开发者需要继承并实现JRCallCallback类中的方法，并通过addCallback接口进行设置。

{c++}
/**
* 添加回调
*
* @param callback JRCallCallback 接口对象
*/
void addCallback(JRCallCallback * callback);

JRCallCallback 所需要实现的方法如下:

{c++}
/**
* 通话状态更新回调
*
* @param item JRCallItem 对象，当 item 为 nil 时表示全部更新
* @param updateType 更新的事件类型
* @see JRCall.UpdateType
*/
void onCallItemUpdated(JRCallItem *item, int updateType);
/**
* 移除通话回调
*
* @param item JRCallItem 对象
* @param reason 通话结束原因
* @see JRCall.TermReason
*/
void onCallItemRemove(JRCallItem *item, int reason);
/**
* 新增通话回调
*
* @param item JRCallItem 对象
*/
void onCallItemAdd(JRCallItem *item);

当收到来电时，通过onCallItemAdd接口进行上报。由对端结束的通话通过onCallItemRemove接口进行通知。通话过程中的各种状态变更，通过onCallItemUpdated接口进行上报。其中onCallItemUpdated接口中的updateType 类型有以下几种情况

{c++}
JRCall::ALERT; // 对方振动铃声
JRCall::RECV_DTMF; // 收到 dtmf 事件
JRCall::OUTGOING; // 通话成功呼出事件
JRCall::TALKING; // 通话成功进行通话

音频设置

有关音频设置的接口见JRMediaDevice.h文件

sdk 控制声音的输入输出

当 setAdpType 设置EN_INPUTSDK_OUTPUTSDK_TYPE类型时， 能够不需要外部输入和输出音频流。

通过调用startAudio()接口进行打开音频， 通过stopAudio()接口进行关闭音频。

{c++}
/**
* @brief 打开音频设备
* 音频的获取由sdk自动控制, 该接口在通话开始前进行调用，若由用户自动传递媒体流则不需要调用该接口
*
* @return 0, 表示接口调用成功， 1表示接口调用失败
*/
int startAudio();
/**
* @brief 关闭音频设备
* 音频的获取由sdk自动控制, 该接口在通话开始后进行调用，若由用户自动传递媒体流则不需要调用该接口音
*
* @return 0. 表示接口调用成功，1表示接口调用失败
*/
int stopAudio();

音箱侧控制音频输入输出的相关定义, 开发者需要通过setAdpType接口设置为EN_INPUTUSER_OUATPUTUSER类型

{c++}
/**
* The audio input data entry to ZMF, each callback will obtain the data.
* Multiple data will mix in the callback of the jssmme Engine,
* and the first input will be main channel.
*
* @param[in] inputId unique name of the audio input
* @param[in] sampleRateHz the sample rating of the pcm data
* @param[in] iChannels the channel number of the pcm data
* @param[in] buf the pcm data
* @param[in] len the pcm data length
* @param[in,out] micLevel
* @param[in] playDelayMS
* @param[in] recDelayMS
* @param[in] clockDrift
*
*/
void onAudioInput(const char *inputId, int sampleRateHz, int iChannels,
unsigned char *buf, int len, int *micLevel,
int playDelayMS, int recDelayMS, int clockDrift);
/**
* tell ZMF the audio input has stopped
*
* @param[in] inputId unique name of the device
*/
void onAudioInputDidStop(const char *inputId);
/**
* The outlet which audio output can get data from.
*
* @param[in] outputId unique name of the audio output
* @param[in] sampleRateHz the sample rating of the pcm data
* @param[in] iChannels the channel number of the pcm data
* @param[in] buf the pcm data to be filled
* @param[in] len the pcm data length
*/
void onAudioOutput(const char *outputId, int sampleRateHz, int iChannels,
unsigned char *buf, int len);
/**
* tell ZMF the audio output has stopped
*
* @param[in] outputId unique name of the device
*/
void onAudioOutputDidStop(const char *outputId);
