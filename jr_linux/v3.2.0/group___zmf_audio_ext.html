详细描述

ZMF音频模块的扩展接口，向外部提供控制音频采集与播放的手段。

◆ ZmfAudioInputCallback

typedef void(* ZmfAudioInputCallback) (void *pUser, const char *inputId, int iSampleRateHz, int iChannels, unsigned char *buf, int len, int *micLevel, int playDelayMS, int recDelayMS, int clockDrift)

定义一种回调函数的类型，该回调函数用于从zmf音频模块获取音频输入数据。

该回调函数将在@ Zmf_OnAudioInput 接口执行时被调用。

参数
[in]pUser通过Zmf_AddAudioInputCallback 接口注册的用户数据
[in]inputId音频输入的Id
[in]iSampleRateHz音频输入数据的采样率，单位为Hz
[in]iChannels音频输入数据的通道数
[in]buf音频输入数据的首地址
[in]len音频输入数据的大小，单位为采样点
[in,out]micLevel系统提供的音频输入数据的麦克风增益值，[0,100]
[in]playDelayMS系统提供的音频播放延时，单位ms
[in]recDelayMS系统提供的音频采集延时，单位ms
[in]clockDrift系统提供的时钟漂移，单位ms
参见
Zmf_AudioInputRemoveCallback
参数
[in,out]micLevel

void Zmf_OnAudioInput ( const char *  inputId,

int *  micLevel,
int  playDelayMS,
int  recDelayMS,
int  clockDrift 
)

zmf音频模块获取音频输入数据的接口。

所有通过Zmf_AudioInputAddCallback 接口注册的回调函数都会在本接口 执行时被调用，从而获取zmf音频模块采集的音频输入数据。如果音频输入数据有多个通道， 将在jssmme引擎中进行混音，第一个输入将会作为主通道。

参数
[in]inputId音频输入的Id
[in]sampleRateHz音频输入数据的采样率，单位为Hz
[in]iChannels音频输入数据的通道数
[in]buf音频输入数据的首地址
[in]len音频输入数据的大小，单位为采样点
[in,out]micLevel系统提供的音频输入数据的麦克风增益值，[0,100]
[in]playDelayMS系统提供的音频播放延时，单位ms
[in]recDelayMS系统提供的音频采集延时，单位ms
[in]clockDrift系统提供的时钟漂移，单位ms

zmf音频模块获取音频输出数据的接口。

所有通过Zmf_AudioOutputAddCallback 接口注册的回调函数都会在本接口 执行时被调用，在指定的缓存地址上进行音频输出数据的填充，从而将数据传递给音频输出设备进行播放。

参数
[in]outputId音频输出的Id
[in]sampleRateHz音频输出数据的采样率，单位为Hz
[in]iChannels音频输出数据的通道数
[in]buf音频输出数据的首地址
[in]len音频输出数据的大小，单位为采样点
